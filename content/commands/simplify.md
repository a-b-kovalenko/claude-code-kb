## 📝 TL;DR

`/simplify` — автоматичний cleanup коду без пошуку багів. Запускає 4 субагенти паралельно і одразу застосовує знайдені покращення. Для пошуку багів — `/code-review`.

## Що перевіряє

Чотири субагенти працюють паралельно, кожен зосереджений на своєму аспекті:

- **Reuse** — чи можна замінити код існуючими хелперами або утилітами.
- **Simplification** — надлишкова складність, дублювання, зайві кроки.
- **Efficiency** — неефективні патерни виконання.
- **Altitude** — чи код знаходиться на правильному рівні абстракції: не занадто низькорівневий і не надто узагальнений.

## Синтаксис

```text
/simplify
/simplify <шлях>
/simplify <PR-номер або посилання>
```

Без аргументу — переглядає поточний uncommitted diff. З аргументом — конкретний файл, директорія або PR.

## Порівняння з `/code-review`

| | `/simplify` | `/code-review` |
| :--- | :--- | :--- |
| Баги | Не шукає | Шукає |
| Cleanup | Так, застосовує автоматично | Так, з `--fix` |
| Субагенти | 4 паралельно | Залежить від рівня |
| Коли використовувати | Після реалізації, перед PR | Перед merge або для аудиту |

## Примітки

- Доступний з v2.1.154. На старіших версіях еквівалентний `/code-review --fix`.
- Зміни застосовуються одразу — перевір diff після виконання.
- Для глибокого ревʼю з багами і cleanup разом: `/code-review --fix`.
