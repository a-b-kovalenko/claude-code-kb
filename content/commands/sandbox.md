## 📝 TL;DR

`/sandbox` вмикає OS-рівневу ізоляцію Bash-команд: Claude може працювати автономно, але фізично не може вийти за визначені межі — читати чужі файли, писати поза проєктом або звертатись до недозволених доменів. Альтернатива постійним permission prompts без ризику.

## Синтаксис

```text
/sandbox
```

Відкриває панель із трьома вкладками: **Mode** (режим авто-дозволу), **Overrides** (fallback поведінка), **Config** (поточні налаштування).

## Навіщо

Без sandbox є два варіанти: або дозволяти кожну Bash-команду вручну, або довіряти агенту сліпо. Sandbox — третій варіант: одноразово визначаєш межі, і ОС їх примусово тримає для команди і всіх дочірніх процесів — навіть при prompt injection у коді, що читається. Це особливо актуально для [headless/CI-сценаріїв](../Headless_Mode.md), де агент працює без нагляду.

## Два режими

| Режим            | Як працює                                                                                       |
| :--------------- | :---------------------------------------------------------------------------------------------- |
| **Auto-allow**   | Sandboxed-команди виконуються без питань. Вихід за межі — через звичайний permission flow.      |
| **Regular**      | Sandbox активний, але дозволи запитуються як завжди.                                            |

## Що ізолюється

### Файлова система

За замовчуванням:

- **Write**: тільки поточна директорія і session temp (`$TMPDIR`)
- **Read**: вся файлова система — включно з `~/.aws/credentials`, `~/.ssh/` (треба явно блокувати через `denyRead`)

Розширити write-доступ: `sandbox.filesystem.allowWrite` у `settings.json`.

### Мережа

Жоден домен не дозволений за замовчуванням. При першому зверненні до нового домену — prompt. Після підтвердження домен запам'ятовується.

Pre-allow доменів: `sandbox.network.allowedDomains`.

## Платформи

| Платформа       | Підтримка | Що потрібно                   |
| :-------------- | :-------- | :---------------------------- |
| macOS           | Так       | Нічого (Seatbelt вбудований)  |
| Linux           | Так       | `bubblewrap` + `socat`        |
| WSL2            | Так       | `bubblewrap` + `socat`        |
| Windows native  | Ні        | Використовувати WSL2          |

## Відомі несумісності

- `docker` — несумісний, додати в `excludedCommands`
- `jest` з watchman — зависає, запускати з `--no-watchman`
- `gh`, `gcloud`, `terraform` на macOS — можуть падати на TLS, додати в `excludedCommands`

## Важливі нюанси

- `Read`, `Edit`, `Write` інструменти Claude **не** проходять через sandbox — тільки Bash.
- Субагенти успадковують sandbox батьківської сесії.
- Fallback: команда, що не виконалась у sandbox, за замовчуванням може спробувати без нього. Щоб заборонити: `"allowUnsandboxedCommands": false`.
- Налаштування sandbox зберігаються у `.claude/settings.local.json` — не потрапляє в git.
