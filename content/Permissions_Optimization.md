## 📝 TL;DR

Кожен запит дозволу перериває роботу агента і вимагає вашої уваги. Мета оптимізації — дозволити все що безпечно автоматично, і заблокувати все що небезпечно назавжди. Скіл `/fewer-permission-prompts` автоматично аналізує транскрипти сесій і генерує готовий allowlist.

## Чому виникають запити дозволу

За замовчуванням Claude Code запитує підтвердження для більшості інструментів: `Bash`, `Edit`, `Write`, MCP-виклики. Будь-яка команда без запису в `allow` — пауза для вас.

Запити бувають двох типів:

- **Безпечні, повторювані** — `git status`, `git diff`, читання файлів. Завжди дозволяєте → треба додати в `allow`.
- **Ризиковані, рідкісні** — `git push`, деструктивні команди. Повинні або потрапити в `deny`, або залишитись під ручним контролем.

## `/fewer-permission-prompts` — автоматичний аналіз

Claude Code має вбудований скіл, що сканує транскрипти минулих сесій, знаходить найчастіші read-only Bash і MCP виклики, і генерує готовий `allow`-список:

```text
/fewer-permission-prompts
```

Скіл додає результат безпосередньо в `.claude/settings.json` поточного проєкту. Запустіть після кількох реальних сесій — список буде точним і пріоритизованим за частотою використання.

## Патерни allowlist: синтаксис

Формат: `"ToolName(glob-pattern)"`. Glob-правила:

| Патерн | Що матчить |
| :--- | :--- |
| `"Bash(git *)"` | Будь-яка команда що починається з `git` |
| `"Bash(./gradlew test*)"` | `./gradlew test`, `./gradlew testUnit` тощо |
| `"Read(/home/user/project/*)"` | Читання будь-якого файлу в директорії |
| `"WebFetch(domain:docs.spring.io)"` | Fetch тільки з конкретного домену |
| `"Bash"` | **Будь-яка** bash-команда — небезпечно |

`*` матчить усе крім `/`. Для рекурсивного матчингу — `**`.

## Практичний allow-список для Java/Spring проєкту

```json
{
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff*)",
      "Bash(git log*)",
      "Bash(git branch*)",
      "Bash(git stash*)",
      "Bash(./gradlew test*)",
      "Bash(./gradlew build*)",
      "Bash(./gradlew classes*)",
      "Bash(./gradlew compileJava*)",
      "Bash(grep *)",
      "Bash(find *)",
      "Bash(ls *)",
      "Bash(cat *)",
      "WebFetch(domain:docs.spring.io)",
      "WebFetch(domain:docs.gradle.org)"
    ],
    "deny": [
      "Bash(git push*)",
      "Bash(git reset --hard*)",
      "Bash(git rebase*)",
      "Bash(rm -rf*)",
      "Bash(./gradlew flyway*)"
    ]
  }
}
```

## Обмеження інструментів субагента (`tools:`)

На рівні `allow`/`deny` в `settings.json` обмежуються глобальні дозволи. На рівні субагента — поле `tools:` у frontmatter обмежує які інструменти агент взагалі може використовувати:

```yaml
---
name: explorer
model: claude-haiku-4-5-20251001
tools:
  - Read
  - Grep
  - Glob
  - Bash
---
```

Explorer без `Edit` і `Write` фізично не може змінити файли — навіть якщо `settings.json` їх дозволяє. Це defense-in-depth: субагент обмежений своєю роллю, а не тільки глобальними правилами.

Якщо `tools:` не вказано — субагент успадковує **всі** інструменти батьківської сесії, включно з MCP.

## Workflow оптимізації

```text
1. Запустіть кілька реальних сесій без allowlist
         ↓
2. /fewer-permission-prompts — скіл аналізує транскрипти
         ↓
3. Переглядайте згенерований список:
   - Знайоме і безпечне → залишити в allow
   - Ризиковане → перенести в deny або прибрати
         ↓
4. Закомітити settings.json
         ↓
5. Повторювати після кожної нової категорії задач
```

## Що НІКОЛИ не повинно бути в `allow`

- `"Bash"` без уточнення — повний доступ до shell.
- `"Bash(git push*)"` — push без підтвердження.
- `"Bash(rm *)"` — видалення без підтвердження.
- `"Bash(*flyway*)"` — міграції БД без підтвердження.
- Будь-що що зачіпає `prod`/`production` хости.

Правило: якщо помилка незворотна або бачима іншим — залишайте під ручним контролем.

## allow vs deny vs хук: шпаргалка вибору

| Умова | Правильний інструмент |
| :--- | :--- |
| Завжди безпечно, не питати | `permissions.allow` |
| Завжди небезпечно, ніколи | `permissions.deny` |
| Блокувати тільки якщо містить `--force` | Хук з умовою |
| Субагент не повинен писати файли | `tools:` у frontmatter |
