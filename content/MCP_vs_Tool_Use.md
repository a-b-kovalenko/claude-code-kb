## 📝 TL;DR

Tool use — механізм Claude API: ваш код визначає інструменти і сам їх виконує. MCP — протокол, що стандартизує як зовнішні сервери надають інструменти (Tools), дані (Resources) та шаблони (Prompts). Для примітиву Tools — MCP використовує tool use під капотом, але знімає з вас відповідальність за виконання.

## Принципова різниця

| | Tool use | MCP |
| :--- | :--- | :--- |
| Хто визначає інструменти | Ви в кожному запиті | MCP-сервер один раз |
| Хто виконує | Ваш код | MCP-сервер |
| Зв'язаність | Тісна (прив'язано до застосунку) | Слабка (незалежний сервер) |
| Повторне використання | В межах одного застосунку | Будь-який MCP-сумісний клієнт |
| Де живе логіка | У вашому коді | В MCP-сервері |

## Tool use — механізм

Ви описуєте інструменти у вигляді JSON-схем прямо в API-запиті. Модель вирішує коли викликати інструмент і повертає структурований `tool_use` блок. **Ваш код** читає цей блок, виконує реальну дію і повертає результат моделі.

```text
Клієнт → API-запит (з описом інструментів)
Модель → tool_use block { "name": "read_file", "input": {...} }
Ваш код → читає файл, повертає вміст
Модель → продовжує відповідь з результатом
```

Інструменти визначаються щоразу заново в кожному запиті. Виконання — повністю на стороні клієнта.

## MCP — протокол

MCP-сервер — окремий процес, що реєструє власні інструменти і сам їх виконує. Claude Code підключається при старті, отримує список інструментів через `ListToolsRequest`, і далі викликає їх через `CallToolRequest`. З точки зору моделі — це ті самі `tool_use` виклики, але виконує їх не ваш код, а MCP-сервер.

```text
MCP-сервер реєструє tool під іменем "query"
Claude Code → Anthropic API (додає prefix: "mcp__postgres__query" у переліку інструментів)
Модель → tool_use { "name": "mcp__postgres__query" } — повертає Claude Code, не серверу
Claude Code → CallToolRequest → MCP-сервер виконує SQL
Claude Code → tool_result → Модель продовжує відповідь
```

> Модель (на серверах Anthropic) не знає про ваш локальний MCP-сервер і не звертається до нього напряму. Claude Code є посередником на обох кінцях.

Детальна механіка протоколу (ListToolsRequest, CallToolRequest, sequence diagram): [🔌 MCP — огляд та визначення](MCP_Overview.md).

## Коли що використовувати

**Tool use напряму** — коли будуєте власний застосунок поверх Claude API і повністю контролюєте виконання. Наприклад: кастомна інтеграція з внутрішньою системою, де немає готового MCP-сервера.

**MCP** — коли хочете підключити готову систему (PostgreSQL, Playwright, файлову систему) або коли інструменти мають бути доступні різним клієнтам без переписування логіки.

У Claude Code вбудовані інструменти (`Read`, `Edit`, `Bash`) реалізовані як tool use всередині Claude Code. Власні розширення можливі **виключно через MCP** — прямий tool use недоступний розробнику для розширення CLI.

**Безпекова межа:** при MCP виконання відбувається в окремому процесі, якому можна обмежити права або ізолювати в контейнері. При прямому tool use ваш застосунок є єдиним бастіоном перевірки вхідних даних.

## Java-аналогія

Tool use — це `interface` який ви реалізуєте самі в кожному застосунку:

```java
// Ви описуєте і виконуєте самі
ToolResult execute(ToolCall call) { ... }
```

MCP — готова бібліотека що реалізує цей `interface` і підключається через стандартний механізм:

```java
// Postgres MCP-сервер вже реалізував все за вас
// Ви просто підключаєте залежність
```

Аналог у Spring-екосистемі: tool use ≈ ручний `HttpURLConnection` з парсингом JSON у кожному контролері; MCP ≈ підключити Spring Cloud OpenFeign клієнт, де транспорт і типізація приховані під капотом.
