## 📝 TL;DR

Dynamic Workflows — офіційна фіча Claude Code (research preview, v2.1.154+): Claude сам пише JavaScript-скрипт оркестрації, а runtime виконує його в ізольованому середовищі, запускаючи десятки субагентів паралельно. Призначено для задач масштабу "500+ файлів" — аудити, великі міграції, багатокутне планування.

## Що це таке

Стандартний Claude Code приймає рішення покроково в рамках однієї сесії. Dynamic Workflows виносить логіку оркестрації в окремий JS-скрипт:

```text
Ви описуєте задачу
      ↓
Claude генерує JS-скрипт оркестрації
      ↓
Runtime виконує скрипт в ізольованому середовищі
      ↓
Скрипт spawns десятки субагентів → збирає результати
      ↓
Ваша сесія отримує зведений звіт
```

Сесія залишається responsive — важка робота виконується поза нею.

## Ключові можливості

- **Масштаб:** десятки–сотні субагентів за один запуск (звичайна оркестрація — кілька за хід).
- **Adversarial verification:** кілька незалежних агентів перевіряють результати один одного перед звітом.
- **Reusable script:** оркестрація кодифікується як скрипт, а не як набір покрокових рішень.
- **Ізоляція:** виконується окремо від основної сесії; основна сесія не блокується.

## Типові use cases

| Задача | Чому саме Workflows |
| :--- | :--- |
| Аудит всього репозиторію (500+ файлів) | Паралельний аналіз не вміщується в одну сесію |
| Велика міграція (наприклад, Spring Boot 2→3) | Кожен модуль — окремий агент з власним контекстом |
| Багатокутне планування | Кілька агентів готують плани незалежно, потім Claude обирає найкращий |
| Cross-verified research | Агент A та агент B перевіряють висновки один одного |

## Як запустити

Фіча в research preview — доступна через UI Claude Code (не через CLI-команду). Вимоги: Claude Code v2.1.154+, платний план (Pro/Team/Enterprise) або Anthropic API.

## ⚠️ Застереження

**Research Preview:** синтаксис, API та поведінка фічі можуть змінитися в будь-якому релізі. Перевіряй офіційну документацію Anthropic перед використанням у критичних пайплайнах — v2.1.154+ є мінімальним порогом, але актуальна версія може відрізнятись.

**Витрати:** запуск десятків–сотень субагентів паралельно може швидко вичерпати бюджет. Перед масштабним запуском:

- Протестуй на малій вибірці (5–10 файлів замість 500+).
- Встанови spending limit у Anthropic Console.
- Використовуй `claude-haiku` для субагентів там, де достатньо його можливостей.

## /loop — окрема, але суміжна фіча

`/loop` вирішує іншу задачу: **повторювати prompt у рамках поточної сесії**.

```text
/loop 5m check if CI is green      ← фіксований інтервал (CronCreate)
/loop check deploy status           ← Claude сам обирає інтервал 1хв–1год (ScheduleWakeup)
```

`/loop` — для polling (CI, деплой, PR). Dynamic Workflows — для масштабної паралельної оркестрації.

## Порівняння з іншими підходами

| Підхід | Масштаб | Хто вирішує наступний крок | Коли |
| :--- | :--- | :--- | :--- |
| Субагенти (`.claude/agents/`) | Кілька за хід | Оркестратор покроково | Задачі в рамках сесії |
| [Динамічна оркестрація субагентів](Dynamic_Orchestration.md) | Кілька за хід | Оркестратор за правилами | Адаптивна логіка в сесії |
| **Dynamic Workflows** | Десятки–сотні | JS-скрипт поза сесією | Великомасштабні задачі |
| `/loop` | 1 агент | Cron або ScheduleWakeup | Polling у рамках сесії |
