## 📝 TL;DR

Агентська розробка — це перехід від використання ШІ як "розумного пошуку" до використання його як автономного партнера, здатного виконувати повний цикл задач: дослідження, реалізація та верифікація.

## Що таке "Агентність"?

На відміну від звичайних LLM-чатів, AI-агент (Claude Code) має **інструменти (Tools)**. Він не просто генерує текст, а може:

- Читати та писати файли.
- Виконувати команди в терміналі.
- Досліджувати структуру проєкту через Grep та Glob.
- Запускати тести та аналізувати їх результати.

Це перетворює AI з "консультанта" на "виконавця".

## Claude API vs Claude Code: Де межа?

Розуміння різниці критичне для вибору правильного інструменту:

| Характеристика | Claude API | Claude Code |
| :--- | :--- | :--- |
| **Ціль** | Будування AI-фіч у вашому продукті | Розробка вашого коду (інструмент інженера) |
| **Взаємодія** | Через HTTP запити / SDK | CLI, десктоп-застосунок, IDE-плагіни (VS Code, JetBrains), веб (claude.ai/code) |
| **Доступ до файлів** | Тільки те, що ви передали в промпт | Повний доступ до файлової системи проєкту |
| **Виконання коду** | Ні (тільки генерація) | Так (через Bash-інструменти) |
| **Оплата** | За токени (per-use) | Підписка (Claude Pro/Team/Enterprise) |

**Висновок:** Якщо ви хочете автоматизувати рефакторинг або написання тестів у своєму проєкті — ваш вибір **Claude Code**.

## Ключові стовпи агентського підходу

### 1. Автономне дослідження (Exploration)

Замість того, щоб копіювати весь код у чат, ви даєте агенту високорівневу задачу ("Знайди, як обробляється Kafka-подія X"). Агент сам знайде потрібні сервіси, DTO та конфігурації, використовуючи інструменти пошуку.

### 2. Замкнений цикл (Closed Loop)

Агент працює за принципом **Plan -> Act -> Verify**. Він не вважає задачу виконаною, поки не запустить тест і не побачить "зелений" результат. Це радикально знижує кількість галюцинацій та помилок у логіці.

### 3. Формалізація контексту (Context as Code)

Ми не пояснюємо агенту правила проєкту щоразу. Ми фіксуємо їх у `CLAUDE.md`, `.claude/rules/` та `SKILL.md`. Це робить знання про архітектуру частиною репозиторію, доступною для будь-якого агента (або нового розробника).

## Чому це вигідно розробнику?

- **Зняття рутини:** Написання boilerplate-коду, міграцій та тестів делегується агенту.
- **Швидкість навчання:** Агент миттєво "входить у контекст" нового модуля, пояснюючи вам, як він працює.
- **Якість:** Агент не лінується запускати тести після кожної маленької правки.
