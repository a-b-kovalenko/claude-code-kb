## 📝 TL;DR

Claude Code Desktop — нативний додаток для Mac і Windows, що додає до CLI повноцінний UI: паралельні сесії з git worktree ізоляцією, вбудований diff-рев'ю з коментарями, моніторинг PR та CI, вбудований браузер для перегляду застосунку. Для CLI-фіч різниці немає — хуки, settings, MCP-сервери працюють однаково.

## Сесії та паралельна робота

Кожна розмова в Desktop — це **сесія** зі своєю історією, папкою проєкту та змінами. Паралельні сесії в git-репозиторії автоматично отримують окремі **git worktrees** — зміни в одній сесії не впливають на інші до комміту.

Worktrees зберігаються в `<project-root>/.claude/worktrees/`. Перемикання між сесіями: `Ctrl+Tab` / `Ctrl+Shift+Tab`. Дві сесії поруч: `Cmd+клік` на сесію в сайдбарі.

## Режими дозволів

Перемикаються в будь-який момент кнопкою біля поля вводу:

| Режим | Поведінка |
| :--- | :--- |
| **Ask permissions** | Питає перед кожним редагуванням і командою. Рекомендовано на старті. |
| **Auto accept edits** | Автоматично приймає зміни файлів, але питає перед командами. |
| **Plan mode** | Тільки досліджує і пропонує план — без змін коду. |
| **Auto** | Виконує все самостійно з фоновими перевірками безпеки. |

## Diff-рев'ю та PR моніторинг

Після змін Claude показує індикатор `+12 -1`. Клік відкриває diff-переглядач: список файлів ліворуч, зміни праворуч. Можна клікнути на будь-який рядок і залишити коментар — Claude враховує їх і вносить правки.

Кнопка **Review code** запускає автоматичне ревʼю: Claude шукає помилки компіляції, логічні баги та вразливості безпеки (не стиль і форматування).

Після відкриття PR зʼявляється **CI status bar** — Desktop опитує результати через `gh` і сповіщає про завершення. Можна увімкнути **Auto-fix** (Claude виправляє фейли) та **Auto-merge** (мердж після зеленого CI).

> Потрібен встановлений і авторизований `gh`:
>
> ```bash
> brew install gh && gh auth login
> ```

## Вбудований браузер (Preview)

Claude може запустити dev-сервер і відкрити вбудований браузер. Після кожного редагування Claude автоматично верифікує зміни: робить скріншоти, інспектує DOM, клікає елементи і виправляє знайдені проблеми.

Підтримує також: статичний HTML, PDF, зображення, відео з проєкту.

## Гнучкий workspace

Панелі drag-and-drop: chat, diff, preview, terminal, file, plan, tasks, subagent — будь-яке розташування. Вбудований термінал (`Ctrl+\``) відкривається в директорії проєкту і бачить ті самі файли що й Claude.

Корисні шорткати (macOS):

| Шорткат | Дія |
| :--- | :--- |
| `Cmd+N` | Нова сесія |
| `Cmd+;` | Side chat (питання без переривання сесії) |
| `Cmd+Shift+D` | Відкрити diff |
| `Cmd+Shift+P` | Відкрити preview |
| `Ctrl+\`` | Відкрити термінал |

## Side chat

`Cmd+;` відкриває бічний чат у контексті поточної сесії — можна поставити питання або уточнити деталь, не перериваючи основну задачу.

## Dispatch — задачі з телефону

**Dispatch** дозволяє надіслати задачу з Claude mobile app і вона автоматично запускає нову Desktop-сесію на твоєму маці. Відрізняється від [Remote Control](Remote_Control.md): там ти керуєш вже запущеною сесією, тут — запускаєш нову дистанційно.

## Scheduled tasks

Desktop підтримує заплановані задачі за розкладом — наприклад, щоденний code review або перевірка залежностей. Налаштовується в Settings → Claude Code.

## Що залишається як у CLI

- Хуки (`settings.json`, `md-lint.sh` тощо) — повністю підтримуються.
- MCP-сервери — ті самі.
- Skills, `/plan`, `/remote-control` — ті самі команди.
- `settings.json` і `settings.local.json` — читаються однаково.
