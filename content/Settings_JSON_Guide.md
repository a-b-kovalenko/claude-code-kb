## 📝 TL;DR

`settings.json` — декларативна конфігурація Claude Code: які інструменти автоматично дозволено, які заблоковано назавжди, які змінні середовища бачить агент та яка модель використовується за замовчуванням. Це альтернатива хукам там, де не потрібна умовна логіка — достатньо статичного списку правил.

## Де живуть файли налаштувань

| Файл | Область | Чи комітити |
| :--- | :--- | :--- |
| `.claude/settings.json` | Проєктний — для всієї команди | Так |
| `.claude/settings.local.json` | Проєктний — тільки локально | Ні (додати в `.gitignore`) |
| `~/.claude/settings.json` | Персональний — усі проєкти | — |

**Пріоритет (від найвищого до найнижчого):** enterprise policy → user settings → project settings → project local.

Якщо параметр присутній у кількох файлах, перемагає файл із вищим пріоритетом. `allow`/`deny`-списки з різних файлів **об'єднуються**, а не замінюють один одного.

## Структура файлу

```json
{
  "permissions": {
    "allow": [...],
    "deny":  [...]
  },
  "env": {
    "VARIABLE": "value"
  },
  "model": "claude-sonnet-4-6",
  "hooks": { ... }
}
```

`hooks` — вже розглянуто окремо в [🔧 Розробка власного хука](Hook_Development_Guide.md).

## Permissions: allow та deny

Записи в `allow` — інструменти, що виконуються **без запиту дозволу**. Записи в `deny` — інструменти, що **ніколи не виконуються** (агент отримує відмову).

### Формат записів

| Запис | Що дозволяє / блокує |
| :--- | :--- |
| `"Bash"` | Будь-яка Bash-команда |
| `"Bash(git *)"` | Bash-команди, що починаються з `git` |
| `"Bash(npm test*)"` | Bash-команди, що починаються з `npm test` |
| `"Read"` | Читання будь-якого файлу |
| `"Read(src/*)"` | Читання файлів у директорії `src/` |
| `"Edit"` | Редагування будь-якого файлу |
| `"Write(*.md)"` | Запис тільки Markdown-файлів |

`deny` перемагає `allow`: якщо одна й та сама команда є в обох списках, вона заблокована.

### Типова конфігурація для Spring Boot проєкту

```json
{
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff*)",
      "Bash(git log*)",
      "Bash(mvn test*)",
      "Bash(mvn compile*)",
      "Bash(./mvnw test*)",
      "Bash(./mvnw compile*)"
    ],
    "deny": [
      "Bash(git push*)",
      "Bash(git reset --hard*)",
      "Bash(rm -rf*)",
      "Bash(mvn flyway:repair*)",
      "Bash(mvn flyway:baseline*)"
    ]
  }
}
```

Часті git-команди для читання — в `allow`, щоб агент не питав дозволу при кожному `git diff`. Деструктивні операції та пуш — в `deny`, щоб виключити їх повністю, навіть якщо агент впевнений у правильності дії.

## env: змінні середовища

Змінні з `env` додаються до оточення кожної Bash-команди агента в цьому проєкті.

```json
{
  "env": {
    "SPRING_PROFILES_ACTIVE": "test",
    "TESTCONTAINERS_RYUK_DISABLED": "true",
    "DOCKER_HOST": "unix:///var/run/docker.sock"
  }
}
```

Типове використання:

- Активний профіль Spring для тестового запуску агентом.
- `TESTCONTAINERS_RYUK_DISABLED=true` — прибирає попередження Testcontainers у CI-подібному середовищі.
- Шлях до Docker socket, якщо нестандартний (наприклад, Colima).

**Секрети** (паролі, токени) — не кладіть у `settings.json`, який комітиться. Використовуйте `settings.local.json` або завантажуйте через `apiKeyHelper`.

## model: модель за замовчуванням

```json
{
  "model": "claude-sonnet-4-6"
}
```

Перевизначає модель тільки для поточного проєкту. Корисно для команди: зафіксувати дешевшу модель для проєктів з великим обсягом рутинних задач. Конкретний агент (в `.claude/agents/*.md`) може перевизначити модель ще раз через власний `model:` у frontmatter.

## allow vs deny vs хук: що обрати

| Задача | Правильний інструмент |
| :--- | :--- |
| Авто-дозволити git read-команди | `permissions.allow` |
| Назавжди заблокувати `git push` | `permissions.deny` |
| Заблокувати `git push --force`, але дозволити звичайний push | Хук (`PreToolUse` + `jq`-перевірка) |
| Попередити агента про важливий файл, але не блокувати | Хук (`exit 0` + stdout) |
| Зафіксувати змінні для Testcontainers | `env` |

Правило вибору: `allow`/`deny` — для статичних, беззаперечних правил. Хук — коли потрібна умовна логіка або зворотний зв'язок без блокування.
