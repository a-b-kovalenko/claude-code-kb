## 📝 TL;DR

Ефективна робота з AI базується на розумінні ієрархії інструкцій (CLAUDE.md) та впровадженні циклу синтезу знань. Замість складного пошуку (RAG), ми використовуємо структурований Markdown, що вантажиться прямо в контекст.

## Навіщо керувати пам'яттю?

AI-агент має обмежене вікно контексту. Якщо не структурувати знання, агент почне "забувати" важливі конвенції проєкту або повторювати одні й ті самі помилки. Наша мета — перетворити сирий досвід взаємодії на **імутабельні правила**, які автоматично підхоплюються кожною новою сесією.

## Ієрархія пам'яті (Claude Code)

Claude Code завантажує інструкції в певному порядку. Розуміння цієї черговості дозволяє правильно перевизначати правила:

1. **Корпоративні політики** (Managed Policies) — глобальні обмеження.
2. **Project Memory (`./CLAUDE.md`)** — головний файл проєкту. **Найважливіша точка контролю.**
3. **Project Rules (`.claude/rules/`)** — модульні правила (наприклад, `jpa.md`, `testing.md`).
4. **User Memory/Rules (`~/.claude/`)** — ваші персональні вподобання на всіх проєктах.
5. **Auto Memory** — динамічний нотатник Claude, куди він записує спостереження під час сесії.

**Порада:** Тримайте `CLAUDE.md` стислим (<60 рядків). Специфіку модулів виносьте в `.claude/rules/`.

## Патерн "Memory як синтез" (Karpathy LLM Wiki)

Замість того, щоб покладатися на RAG (пошук по базі даних), ми використовуємо підхід Андрія Карпаті:

- **Сирі нотатки** (Auto Memory) — це вихідний код.
- **Структурована Wiki** (CLAUDE.md / Rules) — це "скомпільована" програма, готова до виконання агентом.
Ми не запитуємо AI про випадкові факти; ми даємо йому **цілісну, синтезовану картину** проєкту.

## Цикл компіляції знань (Compile Cycle)

Щоб база знань не деградувала в купу розрізнених нотаток, ми проводимо регулярну "компіляцію":

### 1. Collect (Збір)

Auto Memory автоматично збирає ваші `#` спостереження або розмовні зауваження ("пам'ятай, що ми завжди використовуємо X"). Це сирий, неструктурований матеріал.

### 2. Compile (Аналіз)

Раз на спринт попросіть агента: *"Read my memory and rules. Find established patterns, contradictions, and outdated info."* Агент синтезує хаотичні записи у когерентні пропозиції.

### 3. Promote (Затвердження)

Визріле й стабільне → у версійовані файли:

- Загальні правила → `CLAUDE.md`.
- Технічна специфіка → `.claude/rules/` або `SKILL.md`.
Це робить знання доступними для всієї команди через Git.

### 4. Prune (Очищення)

Видалення перенесеного контенту з Auto Memory. Це звільняє місце в контексті та запобігає дублюванню інструкцій.

## Практичні команди

- `# <інструкція>` — швидкий запис в Auto Memory прямо під час розмови. Наприклад: `# Завжди використовуй AssertJ для нових тестів`.
- `/memory` — відкрити файли пам'яті на редагування.
- `/compact` — стиснути історію розмови (контекстні файли перечитуються з нуля).
