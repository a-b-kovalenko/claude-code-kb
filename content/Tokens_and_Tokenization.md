## 📝 TL;DR

Токен — не слово і не символ, а підслівна одиниця BPE-токенізатора. Англійський текст токенізується ефективніше за український: в ~1.5–2 рази менше токенів для еквівалентного змісту. Це впливає на вартість і розмір контекстного вікна.

## Що таке токен

Токен — фрагмент тексту, що є базовою одиницею обробки моделі. Він не збігається ні зі словом, ні з символом:

- Коротке часте слово → зазвичай 1 токен
- Довге або рідкісне слово → кілька токенів
- Пробіл, розділовий знак → окремий токен або частина сусіднього

Модель "бачить" послідовність токенів — не символів і не слів.

## Як це працює: BPE

Claude використовує **Byte Pair Encoding (BPE)**: токенізатор навчається об'єднувати часті пари байтів у спільний токен. Чим частіше послідовність зустрічається в тренувальних даних — тим довший токен вона утворює.

Наслідок: англійська (домінує в тренувальних даних) отримує довші, "жирніші" токени. Кирилиця — коротші, "тонші".

## Англійська: ефективна токенізація

Правило для оцінки: **~4 символи ≈ 1 токен**, ~0.75 слова ≈ 1 токен.

```text
"Hello"           → 1 токен
"function"        → 1 токен
"Hello, world!"   → 4 токени  (Hello / , / world / !)
"The quick brown fox jumps over the lazy dog"
                  → ~9 токенів
```

Загальний промпт у 1 000 слів ≈ 1 300 токенів.

## Українська: менш ефективна токенізація

Кирилиця у UTF-8 займає 2 байти на символ. Оскільки кирилічні послідовності рідше зустрічались у тренуванні, токенізатор ріже їх дрібніше.

Правило для оцінки: **~2 символи ≈ 1 токен**.

```text
"кіт"             → 2–3 токени   (vs "cat" → 1)
"функція"         → 3–4 токени   (vs "function" → 1)
"Привіт, світ!"   → ~8–9 токенів (vs "Hello, world!" → 4)

"Швидка руда лисиця стрибає через ледачого пса"
                  → ~16–18 токенів  (vs ~9 англійською)
```

Загальний промпт у 1 000 українських слів ≈ 2 000–2 600 токенів.

## Практичні наслідки

### Вартість

Якщо промпт або нотатки написані українською — вони коштують у ~1.5–2 рази дорожче за англійський еквівалент. Для великих обсягів (system prompt, CLAUDE.md, контекст агента) це суттєво.

### Контекстне вікно

200k токенів для Sonnet — це ~150k слів англійською, але лише ~75–100k слів українською. При плануванні довгих сесій з україномовним контентом корисно знижувати оцінку вдвічі.

### Оцінка "на льоту"

| Мова | Слів у промпті | Приблизно токенів |
| --- | --- | --- |
| Англійська | 1 000 | ~1 300 |
| Українська | 1 000 | ~2 000–2 600 |
| Код (Python/Java) | 1 000 | ~700–900 |

Код токенізується ефективніше за природну мову — ключові слова та ідентифікатори часто зустрічаються в тренуванні.

## Підрахунок токенів

Точний підрахунок — через API:

```python
import anthropic

client = anthropic.Anthropic()
response = client.messages.count_tokens(
    model="claude-sonnet-4-6",
    messages=[{"role": "user", "content": "Привіт, як справи?"}]
)
print(response.input_tokens)  # → ~9
```

У status line Claude Code — поле `tok:` показує input + output останнього виклику.
