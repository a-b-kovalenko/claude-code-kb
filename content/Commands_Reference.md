## 📝 TL;DR

Claude Code має два рівні команд: **вбудовані** (нативні функції CLI) і **скіли** (розширені можливості, що викликаються через `/`). Цей файл — швидкий довідник по обох категоріях.

## Вбудовані команди

### Сесія і контекст

| Команда | Що робить |
| :--- | :--- |
| `/clear` | Повністю очищує історію розмови. CLAUDE.md і rules перечитуються заново. Ядерний варіант перезапуску. |
| `/compact [інструкція]` | Стискає історію у резюме, звільняє місце в контексті. Без аргументу — авто-резюме; з аргументом — фокусує на важливому. |

### Режими роботи

| Команда | Що робить |
| :--- | :--- |
| `/plan` | Входить у Plan Mode: інструменти запису заблоковані, агент тільки досліджує і проєктує. Виходу — після апруву плану. |
| `/fast` | Перемикає Fast Mode: Opus із прискореним виводом. Вмикає або вимикає за повторним викликом. |

### Налаштування

| Команда | Що робить |
| :--- | :--- |
| `/config` | Відкриває інтерактивне меню конфігурації: тема, модель, базові налаштування. |
| `/memory` | Відкриває файли auto-memory на редагування. Корисно для ручного коригування або видалення застарілих записів. |
| `/init` | Аналізує кодову базу і створює `CLAUDE.md`. Якщо файл вже є — пропонує доповнення. |

### Shell-шорткат

```text
! <команда>
```

Виконує shell-команду прямо в поточній сесії — її stdout потрапляє в контекст розмови. Корисно для команд що потребують інтерактивного логіну (`! gcloud auth login`) або для швидкої перевірки стану без переключення вікна.

## Thinking-префікси

Не slash-команди — просто початок повідомлення. Claude Code визначає рівень бюджету автоматично:

| Префікс | Бюджет |
| :--- | :--- |
| `think about...` | Базовий |
| `think hard about...` | Середній |
| `think harder...` | Глибокий |
| `ultrathink...` | Максимальний |

Префікси розпізнаються тільки англійською. Решта запиту — будь-якою мовою: `think hard: поясни цю залежність...`

Доступні тільки на Sonnet і Opus. Детальніше: [🧠 Extended Thinking та /think](Extended_Thinking.md).

## Скіли (вбудовані, викликаються через `/`)

### Якість коду

| Команда | Що робить |
| :--- | :--- |
| `/code-review [low\|medium\|high\|max\|ultra]` | Ревʼю поточного diff на баги та спрощення. `ultra` — глибокий мультиагентний аналіз у хмарі. `--comment` постить знахідки як inline-коментарі PR; `--fix` — одразу застосовує. |
| `/code-review ultra <PR#>` | Те саме, але для конкретного GitHub PR за номером. |
| `/security-review` | Ревʼю на вразливості: OWASP Top 10, injection, secrets у коді. |
| `/simplify` | Шукає надлишковий код, дублювання, зайві абстракції — і одразу застосовує спрощення. |
| `/verify` | Запускає застосунок і перевіряє що зміна реально працює (golden path + edge cases). |

### Запуск

| Команда | Що робить |
| :--- | :--- |
| `/run` | Запускає проєкт за типом (CLI, сервер, Electron тощо). Спочатку шукає project-skill з інструкцією запуску. |

### PR та ревʼю

| Команда | Що робить |
| :--- | :--- |
| `/review` | Ревʼю pull request: коректність, архітектура, відповідність конвенціям. |
| `/init` | (Також скіл) Ініціалізує `CLAUDE.md` з аналізу кодової бази. |

### Автоматизація

| Команда | Що робить |
| :--- | :--- |
| `/loop [інтервал] <prompt>` | Запускає prompt повторно. З інтервалом (`5m`) — фіксований cron; без — Claude сам обирає паузу. Для polling CI, деплою, PR. |
| `/schedule` | Створює і керує хмарними агентами за розкладом (cron). Для задач що мають виконуватись автономно без відкритої сесії. |

### Конфігурація та довідка

| Команда | Що робить |
| :--- | :--- |
| `/update-config` | Налаштовує `settings.json` через скіл: дозволи, хуки, змінні середовища. Альтернатива ручному редагуванню. |
| `/fewer-permission-prompts` | Сканує транскрипти сесій, знаходить повторювані безпечні виклики і додає їх у `allow`-список `settings.json`. Детальніше: [🔐 Оптимізація дозволів](Permissions_Optimization.md). |
| `/keybindings-help` | Допомагає кастомізувати клавіатурні скорочення в `~/.claude/keybindings.json`. |
