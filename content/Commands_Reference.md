## 📝 TL;DR

Швидкий довідник по всіх типах команд Claude Code:

- **Вбудовані команди** — нативні функції CLI (`/clear`, `/plan`, `/config`).
- **Slash-команди Claude Code** — вбудовані скіли платформи (`/code-review`, `/verify`, `/simplify`); викликаються через `/`.
- **Власні скіли** — директорії `.claude/skills/<name>/SKILL.md` або `~/.claude/skills/<name>/SKILL.md`; викликаються через `/name` або завантажуються автоматично за темою.
- **Кастомні команди** (`.claude/commands/*.md`) — старий flat-file формат; досі працює, але скіли мають пріоритет при збігу імені.

## Вбудовані команди

### Сесія і контекст

| Команда | Що робить |
| :--- | :--- |
| `/clear [назва]` | Починає нову розмову з порожнім контекстом. Попередня сесія зберігається і доступна через `/resume`. Аргумент — мітка для зручності пошуку. Аліаси: `/reset`, `/new`. |
| `/compact [інструкція]` | Стискає історію у резюме, звільняє місце в контексті. Без аргументу — авто-резюме; з аргументом — фокусує на важливому. |
| `/resume [сесія]` | Відкриває picker попередніх сесій або відновлює конкретну за ID чи назвою. Background-сесії позначені `bg`. Аліас: `/continue`. |
| `/branch [назва]` | Створює гілку поточної розмови — можна спробувати інший напрямок, не втрачаючи оригінал. Повернутись до оригіналу — через `/resume`. |
| `/export [файл]` | Зберігає поточну розмову як plain text. З аргументом — записує у файл; без — відкриває діалог для копіювання в буфер або збереження. |

### Режими роботи

| Команда | Що робить |
| :--- | :--- |
| `/plan` | Входить у Plan Mode: інструменти запису заблоковані, агент тільки досліджує і проєктує. Виходу — після апруву плану. |
| `/fast [on\|off]` | Вмикає Fast Mode: Opus до 2.5x швидший при вищій ціні за токен. Автоматично перемикає на Opus якщо активна інша модель. Активний режим — іконка `↯` поруч з промптом. Вимкнення не повертає попередню модель — для зміни: `/model`. Research preview, тільки Anthropic API/підписка. |

### Налаштування

| Команда | Що робить |
| :--- | :--- |
| `/config` | Відкриває інтерактивне меню конфігурації: тема, модель, базові налаштування. |
| `/memory` | Відкриває файли auto-memory на редагування. Корисно для ручного коригування або видалення застарілих записів. |
| `/init` | Аналізує кодову базу і створює `CLAUDE.md`. Якщо файл вже є — пропонує доповнення. |

### Shell-шорткат

```text
! <команда>
```

Виконує shell-команду прямо в поточній сесії — її stdout потрапляє в контекст розмови. Корисно для команд що потребують інтерактивного логіну (`! gcloud auth login`) або для швидкої перевірки стану без переключення вікна.

## Thinking-префікси

Не slash-команди — просто початок повідомлення. Claude Code визначає рівень бюджету автоматично:

| Префікс | Бюджет |
| :--- | :--- |
| `think about...` | Базовий |
| `think hard about...` | Середній |
| `think harder...` | Глибокий |
| `ultrathink...` | Максимальний |

Префікси розпізнаються тільки англійською. Решта запиту — будь-якою мовою: `think hard: поясни цю залежність...`

Доступні тільки на Sonnet і Opus. Детальніше: [🧠 Extended Thinking та /think](Extended_Thinking.md).

## Slash-команди Claude Code

### Якість коду

| Команда                                        | Що робить                                                                                                                                                 |
| :--------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/code-review [low\|medium\|high\|max\|ultra]` | Ревʼю diff на баги та спрощення. Рівні: low→ultra. `--fix` — застосувати; `--comment` — PR-коментарі. Детально: [/code-review](commands/code-review.md).  |
| `/code-review ultra <PR#>`                     | Те саме, але для конкретного GitHub PR за номером.                                                                                                        |
| `/security-review`                             | Ревʼю на вразливості: OWASP Top 10, injection, secrets у коді.                                                                                            |
| `/simplify`                                    | Шукає надлишковий код, дублювання, зайві абстракції — і одразу застосовує спрощення. 4 субагенти паралельно. Детально: [/simplify](commands/simplify.md). |
| `/verify`                                      | Запускає застосунок і перевіряє що зміна реально працює (golden path + edge cases).                                                                       |

### Запуск

| Команда | Що робить |
| :--- | :--- |
| `/run` | Запускає проєкт за типом (CLI, сервер, Electron тощо). Спочатку шукає project-skill з інструкцією запуску. |

### PR та ревʼю

| Команда        | Що робить                                                                                                                                                                                                                                                        |
| :------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/review [PR]` | Глибший **read-only** аналіз PR локально: коректність, архітектура, відповідність конвенціям. Змін не вносить — тільки звіт. Без аргументу — поточний стан; з номером PR — конкретний pull request. Для хмарного мультиагентного аналізу — `/code-review ultra`. |

### Автоматизація

| Команда                     | Що робить                                                                                                                                                                                                       |
| :-------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/batch <інструкція>`       | Масштабний паралельний рефакторинг: декомпозує задачу на 5–30 одиниць, показує план, запускає субагент у worktree для кожної і відкриває PR. Детально: [/batch](commands/batch.md).                             |
| `/loop [інтервал] <prompt>` | Запускає prompt повторно. З інтервалом (`5m`) — фіксований cron; без — Claude сам обирає паузу. Детально: [Loop Engineering](Loop_Engineering.md).                                                              |
| `/goal [умова\|clear]`      | Ставить мету: Claude працює самостійно між повідомленнями поки умова не виконана. Без аргументу — показує поточну мету. `clear`/`stop`/`cancel` — скасувати. Детально: [Loop Engineering](Loop_Engineering.md). |
| `/schedule`                 | Створює і керує хмарними агентами за розкладом (cron). Для задач що мають виконуватись автономно без відкритої сесії. Детально: [⚙️ Dynamic Workflows](Claude_Code_Dynamic_Workflows.md).                       |

### Конфігурація та довідка

| Команда | Що робить |
| :--- | :--- |
| `/update-config` | Налаштовує `settings.json` через скіл: завантажує **повну схему** settings.json у контекст. Корисний не лише для змін, а й для **відкриття** маловідомих полів — наприклад, `plansDirectory`. Детально: [⚙️ Конфігурація settings.json](Settings_JSON_Guide.md). |
| `/fewer-permission-prompts` | Сканує транскрипти сесій, знаходить повторювані безпечні виклики і додає їх у `allow`-список `settings.json`. Детальніше: [🔐 Оптимізація дозволів](Permissions_Optimization.md). |
| `/keybindings-help` | Допомагає кастомізувати клавіатурні скорочення в `~/.claude/keybindings.json`. |
| `/statusline` | Налаштовує рядок статусу через `statusline-setup` агента. Скрипт у `~/.claude/statusline-command.sh` має доступ до `$CLAUDE_*` змінних сесії. Детально: [/statusline](commands/statusline.md). |
| `/sandbox` | Вмикає OS-рівневу ізоляцію Bash-команд: write тільки у поточній директорії, мережа — тільки дозволені домени. Альтернатива permission prompts без ризику. Детально: [/sandbox](commands/sandbox.md). |
| `/insights` | Генерує звіт по сесіях: які частини кодової бази зачіпались, патерни взаємодії, friction points. Корисно для рефлексії над власним workflow. |

## Кастомні команди

Директорії у `.claude/skills/<name>/` (проєктні) або `~/.claude/skills/<name>/` (персональні) — доступні як `/name`. Старий формат `.claude/commands/*.md` також підтримується.

| Команда         | Де живе                                  | Що робить                                                                                               |
| :-------------- | :--------------------------------------- | :------------------------------------------------------------------------------------------------------ |
| `/handoff`      | `~/.claude/skills/handoff/SKILL.md`      | Створює handoff-документ для передачі контексту між сесіями. Детально: [/handoff](commands/handoff.md). |
| `/init-project` | `~/.claude/skills/init-project/SKILL.md` | Bootstrap нового або retrofit існуючого проєкту. Детально: [/init-project](commands/init-project.md).   |

Детальніше про створення власних команд: [⚡ Кастомні Slash-команди](Custom_Slash_Commands.md).
