## 📝 TL;DR

Remote Control дозволяє продовжити поточну Claude Code сесію з телефону або будь-якого браузера. Сесія залишається запущеною на твоєму комп'ютері — телефон це лише вікно в неї. Всі локальні інструменти, MCP-сервери та хуки залишаються доступними.

## Як запустити

Три способи — результат однаковий:

```bash
# Новий інтерактивний сеанс одразу з Remote Control
claude --remote-control

# Серверний режим (чекає підключень, без локального вводу)
claude remote-control
```

Або з поточної сесії командою:

```text
/remote-control
```

Після запуску Claude показує QR-код і URL сесії. Скануй QR-код у Claude mobile app або відкрий `claude.ai/code` у браузері.

## Підключення з телефону

Два варіанти:

- **QR-код** — натисни пробіл (в серверному режимі) або відкрий статус-панель стрілкою вниз → Enter.
- **claude.ai/code** — відкрий у браузері, знайди сесію в списку. Remote Control сесії мають іконку комп'ютера із зеленою точкою.

У Claude mobile app: вкладка **Code** → список активних сесій.

## Що залишається доступним

- Весь локальний filesystem
- MCP-сервери (Postgres, Kafka тощо)
- Хуки та `settings.json`
- Autocomplete `@файл` з локального проєкту
- Паралельна робота: можна писати і з терміналу, і з телефону одночасно

## Push-нотифікації

Коли Remote Control активний, Claude може надсилати push на телефон — наприклад, коли довга задача завершилась або потрібне твоє рішення.

Налаштування через `/config` → **Push when Claude decides** та **Push when actions required**.

Вимоги: Claude mobile app встановлено, авторизовано тим самим акаунтом, нотифікації дозволено.

## Вимоги

- План **Pro, Max, Team або Enterprise** — Free не підтримується.
- Авторизація через **claude.ai OAuth** (`/login`), не API-ключ.
- Claude Code **v2.1.51** або новіший (`claude --version`).

## Обмеження

- **Мак має залишатись увімкненим.** Локальний процес `claude` повинен працювати. Мак може бути заблокований, але не в сплячому режимі — інакше процес завершиться і сесія обірветься.
- Один Remote Control на один інтерактивний процес. Для кількох паралельних сесій — серверний режим (`claude remote-control --spawn worktree`).
- Мережевий збій понад ~10 хвилин → сесія тайм-аутиться.

## Remote Control vs Claude Code on the web

| | Remote Control | Claude Code on the web |
| :--- | :--- | :--- |
| Де виконується | На твоїй машині | Хмара Anthropic |
| Локальні файли | Так | Ні (лише GitHub repo) |
| MCP-сервери | Так | Ні |
| Мак має бути увімкнений | Так | Ні |
| Коли використовувати | Продовжуєш локальну роботу з іншого пристрою | Задача без локального середовища |
