## 📝 TL;DR

Скіли та MCP — це способи надання агенту вузькоспеціалізованих знань та прямого доступу до інфраструктури. Вони дозволяють AI працювати не з "текстом коду", а з реальною архітектурою та даними проєкту. MCP-сервери для команди реєструються у `.mcp.json` у корені проєкту.

## Agent Skills: Експертиза за запитом

Скіли (`SKILL.md`) — це інструкції, які завантажуються **тільки тоді**, коли вони потрібні (Progressive Disclosure). Це запобігає засміченню контексту загальними правилами.

### Коли варто створювати скіл?

Скіл вартий існування, якщо він фіксує патерни, де AI систематично помиляється.

- **Persistence (JPA/Hibernate):** Запобігання N+1 SELECT, правильна пагінація, межі транзакцій. Без скіла агент часто пропонує наївні рішення, що вбивають продуктивність.
- **Spring Layering:** Чіткий розподіл між DTO, Entity та Service. Правила використання `@Value`, `@ConfigurationProperties` тощо.
- **Тестування:** Ваші специфічні конвенції (наприклад, обов'язковий DbRider для інтеграційних тестів).

## MCP: Прямий доступ до "заліза"

Model Context Protocol (MCP) дозволяє агенту перестати "вгадувати" стан системи по коду та почати бачити реальність.

### Найцінніші сервери для Backend

- **Postgres MCP:** Агент може прочитати схему таблиць, індекси та навіть приклади даних. Це критично при написанні складних SQL-запитів або міграцій Flyway.
- **Kafka MCP:** Інспекція топіків, перегляд форматів повідомлень та статусів consumer groups.
- **GitHub MCP:** Пряма взаємодія з тікетами, PR та рев'ю без перемикання в браузер.

### Реєстрація серверів

Сервери реєструються у трьох скоупах:

| Скоуп | Файл | Видимість |
| :--- | :--- | :--- |
| `local` *(за замовчуванням)* | `~/.claude.json` | Тільки ви, цей проєкт |
| `project` | `.mcp.json` у корені проєкту | Вся команда, комітити в Git |
| `user` | `~/.claude.json` | Тільки ви, всі проєкти |

```bash
# Додати командний сервер (потрапляє у .mcp.json)
claude mcp add --scope project --transport http postgres https://mcp.postgres.example.com
```

⚠️ **Безпека:** Завжди використовуйте **read-only** облікові записи для MCP-серверів, що дивляться в бази даних. Автономний агент із write-доступом до БД — це ризик.

## Плагіни: Пакування та Дистрибуція

Плагін — це не новий тип інструмента, а **контейнер**. Він може бандлити всередині кілька скілів, субагентів та хуків.

- **Проєктні плагіни:** Живуть у репозиторії, версіюються в Git і доступні всій команді. Це ідеальний спосіб розповсюдження внутрішніх стандартів компанії.
- **Маркетплейс:** Дозволяє встановлювати готові рішення (наприклад, `code-review` плагін від Anthropic).

## Коли що використовувати? (Decision Matrix)

| Інструмент | Ціль | Коли використовувати |
| :--- | :--- | :--- |
| **Rule (`.md`)** | Загальне правило | Вимоги, що діють завжди (наприклад, "тільки Java 21"). |
| **Skill (`SKILL.md`)** | Вузька експертиза | Складні технічні патерни (JPA, Security, Kafka). |
| **Hook (`bash/js`)** | Жорсткий контроль | Детерміновані перевірки (форматування, блок DROP TABLE). |
| **MCP** | Доступ до даних | Коли агенту треба бачити стан БД, черг або зовнішніх API. |
