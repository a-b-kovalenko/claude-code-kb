## 📝 TL;DR

Code RAG (Retrieval-Augmented Generation) та Knowledge Graph — це архітектурний паттерн для роботи з монорепозиторіями на сотні тисяч рядків коду. Він дозволяє AI-агенту знаходити релевантні фрагменти коду та розуміти зв'язки між компонентами, не завантажуючи весь проєкт у контекстне вікно. Весь стек працює локально, забезпечуючи приватність та швидкість.

## Чому стандартного контексту недостатньо?

Навіть із сучасними LLM (Claude 4.6 Sonnet, GPT-5), що мають великі контекстні вікна (200k+ токенів), великі проєкти не вміщаються цілком.

- **Втрата фокусу (Lost in the middle):** Моделі гірше працюють з інформацією в середині великого контексту.
- **Вартість та швидкість:** Передача всього репозиторію на кожний запит є дорогою та повільною.
- **Складні зв'язки:** Код — це не просто текст, це граф залежностей (виклики функцій, ієрархія класів).

## Технічний стек (Local-first)

| Компонент | Інструмент | Роль |
| :--- | :--- | :--- |
| **Parsing** | **tree-sitter** | Інкрементальний парсинг коду, виділення символів (функції, класи, імпорти). |
| **Embeddings** | **Ollama (mxbai-embed-large)** | Перетворення коду у вектори для семантичного пошуку. |
| **Vector DB** | **ChromaDB / Qdrant** | Зберігання векторів та метаданих фрагментів коду. |
| **Graph DB** | **FalkorDB / Memgraph** | Зберігання структурних зв'язків (хто кого викликає). |
| **Orchestration** | **MCP-сервер** | Протокол, через який Claude Code отримує доступ до RAG-інструментів. |

## Архітектура рішення

### 1. Індексація (Indexing)

1. **Скан:** Обхід репозиторію та виділення файлів.
2. **Chunking (Smart):** Замість нарізки по символах, використовується **tree-sitter** для виділення логічних блоків (тіло функції, визначення класу).
3. **Embedding:** Генерація векторів для кожного чанку через Ollama.
4. **Graph Building:** Створення зв'язків між чанками (наприклад, `Function A` -> `Calls` -> `Function B`).

### 2. Пошук (Retrieval)

1. **Query Embedding:** Питання користувача перетворюється у вектор.
2. **Hybrid Search:**
    - **Vector Search:** Знаходить семантично схожий код (наприклад, "де обробляються платежі?").
    - **Keyword Search (BM25):** Знаходить точні збіги назв методів чи класів.
    - **Graph Traversal:** Якщо знайдено функцію, RAG підтягує також її залежності або інтерфейси, які вона реалізує.

### 3. Генерація (Generation)

Знайдений контекст вставляється в системний промпт агента як `relevant_code_snippets`.

## Реалізація через MCP

Найефективніший спосіб інтеграції з Claude Code — створення MCP-сервера, який надає такі інструменти (Tools):

- `search_code(query)` — семантичний пошук по вектору.
- `get_definitions(symbol)` — пошук визначення через граф.
- `get_references(symbol)` — пошук усіх викликів методу/класу.

## Переваги локального підходу

- **Безпека:** Код ніколи не залишає вашу машину.
- **Вартість:** Нульові витрати на API ембеддінгів.
- **Швидкість:** Мінімальна затримка при зверненні до локальної БД.
