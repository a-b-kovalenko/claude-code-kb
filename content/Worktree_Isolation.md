## 📝 TL;DR

Worktree isolation — параметр `isolation: "worktree"` в Agent-виклику, що запускає субагента у відгалуженій копії репозиторію. Основний working tree залишається незайманим. Якщо агент нічого не змінив — worktree прибирається автоматично; якщо змінив — повертає шлях і назву гілки для ревʼю і злиття.

## Що таке git worktree

Git дозволяє тримати кілька робочих директорій з одного репозиторію одночасно — кожна на своїй гілці. `isolation: "worktree"` використовує цей механізм: агент отримує ізольовану копію файлової системи проєкту і не може зіпсувати ваш поточний стан роботи.

## Як запустити

```text
Agent({
  description: "Large-scale refactor of payment module",
  isolation: "worktree",
  prompt: "Refactor PaymentService to use the new event-driven pattern..."
})
```

Параметр `isolation: "worktree"` доступний у виклику Agent-інструменту. У Claude Code CLI це відповідає запуску субагента через `EnterWorktree` / `ExitWorktree`.

## Lifecycle

```text
1. Claude Code створює git worktree у тимчасовій директорії
         ↓
2. Субагент виконує задачу в ізоляції
         ↓
3a. Агент не зробив змін → worktree прибирається автоматично
3b. Агент зробив зміни  → повертається { path, branch }
         ↓
4. Ви ревʼюєте diff, мерджите або відкидаєте
```

Результат при наявності змін:

```text
Worktree created at: /private/var/folders/.../worktree-abc123
Branch: worktree/refactor-payment-1718123456
```

## Коли використовувати

| Сценарій | Чому worktree |
| :--- | :--- |
| Великий рефакторинг (10+ файлів) | Агент може зламати щось несподіване — ізоляція дає час на ревʼю |
| Паралельне дослідження двох підходів | Два агенти в різних worktree → порівнюєте diff і обираєте |
| Ризикована зміна (міграція БД, API-брейкінг) | Зміни не потрапляють у ваш working tree без явного злиття |
| Автономна задача без нагляду | Агент працює поки ви займаєтесь іншим — жодного ризику для поточного стану |

## Коли НЕ потрібно

- Дрібні зміни (один файл, явний локальний контекст) — overhead без користі.
- Коли зміни мають бути негайно видимі в основному дереві (наприклад, генерація конфігу для подальшої роботи).
- Якщо репозиторій не ініціалізований git — worktree вимагає git.

## Злиття результату

Після того як агент повернув `{ path, branch }`:

```bash
# Переглянути що зробив агент
git diff main...worktree/refactor-payment-1718123456

# Злити якщо задоволені
git merge worktree/refactor-payment-1718123456

# Або відкрити PR через gh
gh pr create --head worktree/refactor-payment-1718123456
```

Worktree-гілку можна видалити після злиття:

```bash
git branch -d worktree/refactor-payment-1718123456
```
