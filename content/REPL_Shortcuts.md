## 📝 TL;DR

REPL-шорткати — це префікси і клавіші, що змінюють режим взаємодії з Claude Code прямо в терміналі. Більшість розробників знають лише slash-команди та Enter, але ці шорткати суттєво прискорюють роботу і дають контроль над сесією.

## Префікси повідомлень

Вводяться на початку повідомлення замість звичайного тексту.

| Префікс | Що робить |
| :--- | :--- |
| `#` | Memory mode: зберігає спостереження в auto-memory замість надсилання запиту |
| `@` | Вставляє конкретний файл у контекст без пошуку агентом |
| `!` | Виконує shell-команду; stdout потрапляє в контекст розмови |
| `think ...` / `ultrathink ...` | Вмикає extended thinking з різним бюджетом токенів |

Докладніше про `!` і thinking-префікси: [📖 Довідник команд](Commands_Reference.md).

### `#` — Memory mode

```text
# remember that we always use AssertJ, never assertEquals
```

Claude зберігає спостереження в auto-memory файли (`~/.claude/projects/.../memory/`), а не відповідає на запит. Записи накопичуються між сесіями і підвантажуються автоматично. Щоб перенести їх у `CLAUDE.md` — окремий крок ("compile" у патерні Карпаті).

Детальніше: [💾 Auto-memory система](Auto_Memory_System.md).

### `@` — File mention

```text
@ виправ логіку у src/main/java/com/app/service/OrderService.java
```

Файл одразу потрапляє в контекст. Альтернатива пошуку агентом — точніше і дешевше за токенами, коли ви самі знаєте потрібний файл.

## Клавіатурні шорткати

| Клавіша | Що робить |
| :--- | :--- |
| `Escape` | Зупиняє Claude посеред відповіді |
| `Escape` → `Escape` | Перемотування розмови: показує всі попередні повідомлення, дозволяє стрибнути до будь-якої точки |
| `Escape` → `#` | Зупинити Claude + одразу зафіксувати спостереження в auto-memory |
| `Shift+Tab` → `Shift+Tab` | Входить у Plan Mode: агент досліджує і планує, але не записує файли |
| `Ctrl+V` | Вставити скріншот (на macOS — саме `Ctrl`, не `Cmd`) |
| `Shift+Enter` | Новий рядок без надсилання повідомлення |
| `↑` / `↓` | Навігація по історії введених повідомлень |

### Escape — зупинка з можливістю уточнення

Натискання `Escape` перериває поточну відповідь Claude. Три типових сценарії після зупинки:

**Уточнити запит** — Claude зрозумів неправильно; зупиняєте, перефразовуєте, надсилаєте знову. Це дешевше за очікування повної відповіді і виправлення постфактум.

**Pause перед записом** — у Plan Mode Claude спочатку створює `TaskCreate`-виклики. Між останнім Task і першим `Edit` є природна пауза, де можна натиснути `Escape` і переглянути план до того, як агент почне змінювати файли.

**Ланцюжок Escape → #** — зупинити і одразу зафіксувати спостереження (див. нижче).

### Escape + # — виправлення поведінки на льоту

Сценарій: Claude повторює одну й ту ж помилку. Замість того щоб чекати кінця відповіді:

1. `Escape` — зупинити
2. `# never use H2 in tests, always Testcontainers` — зафіксувати спостереження
3. Повторити запит

Спостереження потрапляє в auto-memory і підвантажується в наступних сесіях. Для негайного ефекту в поточній сесії — також оновіть `CLAUDE.md` вручну або через `/memory`.

### Double Escape — хірургічний відкат

Показує список усіх попередніх повідомлень сесії. Можна вибрати точку і продовжити звідти, пропустивши невдале відгалуження.

Коли що обирати:

| Ситуація | Інструмент |
| :--- | :--- |
| Хочу повернутись до конкретного повідомлення | `Double Escape` |
| Контекст захаращений, але задача ще в процесі | `/compact` |
| Задача змінилась, починаю новий напрямок | `/clear` |

`Double Escape` — єдиний спосіб відкатитись без втрати контексту або його стиснення. Зміни у файлах, зроблені в "відкинутій" гілці, залишаються — git для відкоту коду.

### Shift+Tab × 2 — Plan Mode

Вхід у режим, де Claude **читає і планує, але не пише файли**. Корисно:

- Перевірити, чи правильно агент зрозумів задачу, до того як він почне редагувати код
- Отримати детальний план для складного рефакторингу і скоригувати scope
- Поставити уточнюючі питання без ризику передчасних змін

Вихід з Plan Mode — повторний `Shift+Tab` або явна команда почати реалізацію.

### Ctrl+V — вставка скріншоту (macOS)

На macOS стандартний `Cmd+V` **не працює** для вставки зображень у Claude Code. Використовуйте `Ctrl+V`. Після вставки скріншот стає частиною контексту — Claude бачить зображення і може описати баг, UI-проблему або стектрейс з екрану.
