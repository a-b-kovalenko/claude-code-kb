## 📝 TL;DR

Claude Code доступний у чотирьох режимах: CLI, десктоп, вебзастосунок і IDE-розширення (VS Code, JetBrains). IDE-інтеграція додає inline diff, доступ до діагностики редактора і контекст поточного файлу — без переключення між вікнами.

## Чотири режими Claude Code

| Режим | Коли обирати |
| :--- | :--- |
| **CLI** (`claude`) | Автономні агенти, хуки, довгі сесії, налаштування settings.json |
| **Десктоп** (Mac/Windows) | Те саме що CLI, але з GUI-оберткою |
| **Веб** (claude.ai/code) | Швидкий доступ без встановлення, базові задачі |
| **IDE-розширення** | Повсякденне кодування, inline diff, діагностика |

## IDE-розширення: що додають

### Inline diff

Зміни від агента з'являються безпосередньо в редакторі з підсвіткою додавань/видалень — замість перегляду diff у терміналі. Можна прийняти або відхилити блок змін прямо в IDE.

### Контекст поточного файлу

IDE знає який файл відкритий — агент отримує цей контекст автоматично. Не треба явно вказувати шлях або копіювати код у чат.

### Діагностика редактора

Через MCP-інструмент `getDiagnostics` агент може прочитати поточні помилки і попередження компілятора або лінтера:

```text
Fix all errors in the current file.
```

Агент отримає список діагностик (файл, рядок, повідомлення) і виправить їх з повним розумінням контексту — не по тексту помилки, а по реальному стану проєкту.

### Клавіатурні скорочення в IDE-контексті

Скорочення Claude Code працюють нативно в IDE — не треба перемикатися в термінал щоб відправити запит або підтвердити дію.

## VS Code vs JetBrains

Обидва розширення мають ті самі ключові функції. Різниця — в рідному відчутті: JetBrains-плагін краще інтегрується з IntelliJ-специфічними речами (Run/Debug конфігурації, структура проєкту), VS Code — з його власною екосистемою розширень.

Для Java/Spring Boot розробника на IntelliJ IDEA — JetBrains-плагін є природним вибором.

## Коли CLI переважає IDE

- **Хуки** — PreToolUse/PostToolUse/Stop налаштовуються і надійніше працюють у CLI-сесії.
- **Довгі автономні задачі** — агент працює поки ви займаєтесь іншим; у CLI він не прив'язаний до відкритого вікна IDE.
- **Налаштування settings.json** — CLI дає повний контроль над дозволами і змінними середовища.
- **Worktree isolation** — ізольований запуск субагентів природніше в CLI-контексті.

## Коли IDE переважає CLI

- **Дрібні правки під час кодування** — не виходячи з редактора.
- **Виправлення помилок компілятора** — `getDiagnostics` дає агенту точний список помилок.
- **Рефакторинг з inline review** — ви бачите зміни в контексті коду, а не як текстовий diff.
- **Pair programming** — агент як колега поруч, а не як окремий інструмент.
