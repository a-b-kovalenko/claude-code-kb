## 📝 TL;DR

Домен (20%): конфігурація Claude Code як системи. Або знаєш де лежать файли і що вони роблять — або ні. Найпоширеніша помилка: командні стандарти у user-level config замість project-level.

## CLAUDE.md — ієрархія

```text
~/.claude/CLAUDE.md          ← user-level (особисті налаштування)
<project>/CLAUDE.md          ← project-level (командні стандарти)
<project>/.claude/rules/*.md ← path-specific rules (glob-прив'язані)
```

**Правило:** все що стосується команди та проєкту — у `<project>/CLAUDE.md`. User-level — лише особисті преференції конкретного розробника.

**Типова помилка:** розробник кладе командні конвенції у `~/.claude/CLAUDE.md` — інші члени команди їх не бачать.

## Path-specific Rules (glob patterns)

Файли у `.claude/rules/` прив'язуються до шляхів через glob:

```yaml
# .claude/rules/tests.md
# globs: ["**/*.test.ts", "**/*.spec.ts"]
```

Правило завантажується тільки коли агент працює з файлами що підпадають під glob.

**Типова плутанина:** `**/*.ts` vs `**/*.{ts,tsx}` — другий варіант не працює у більшості реалізацій; треба два окремі glob-патерни.

## Plan Mode vs. Direct Execution

| | Plan Mode | Direct Execution |
| --- | --- | --- |
| Коли | Нетривіальні зміни, невідомий scope | Відомі точкові зміни |
| Що робить | Досліджує → пише план → чекає | Виконує без зупинки |
| Ризик | Повільніше | Може зробити зайве |
| Як увімкнути | `/plan` або `--plan` | За замовчуванням |

**Правило для іспиту:** якщо сценарій описує "агент зробив зайве" — рішення це plan mode або вужчий scope у промпті.

## CI/CD інтеграція

Ключові флаги для non-interactive запуску:

```bash
claude --print          # виводить результат у stdout, не запускає REPL
claude --no-interactive # явно вимикає інтерактивний режим
claude --output-format json  # структурований вивід для парсингу
```

**Типова помилка:** запуск без `--print` у CI — агент чекає інтерактивного вводу і зависає.

## Production Failure Patterns

### 1. Командні стандарти у user-level config

Розробник налаштував все локально — при onboarding нового члена команди нічого не працює як очікується.

**Виправлення:** audit `~/.claude/CLAUDE.md` і перенести командні правила у `<project>/CLAUDE.md`.

### 2. Занадто широкий glob у rules

`**/*` замість конкретного патерну — правило завантажується для всіх файлів, навіть нерелевантних, що засмічує контекст.

**Виправлення:** максимально вузький glob — тільки файли де правило реально потрібне.

### 3. CI зависає через відсутність --print

Pipeline запускає `claude "зроби X"` без флагів — агент чекає людини.

**Виправлення:** завжди `--print` або `--no-interactive` у CI/CD pipeline.
