## 📝 TL;DR

`claude agents` — окремий CLI-інтерфейс (Agent View) для керування фоновими сесіями. Показує всі сесії згруповані за станом, дозволяє відповідати не заходячи в кожну і запускати нові задачі. Альтернативна точка входу замість `claude`.

## Синтаксис

```bash
claude agents                # всі фонові сесії
claude agents --cwd ~/proj   # тільки сесії конкретного проєкту
```

## Стани сесій

Сесії згруповані за станом:

- **Needs input** — чекає на твою відповідь
- **Working** — виконує задачу
- **Completed** — завершила

Іконки у рядку:

| Іконка | Що означає |
| :----- | :--------- |
| `✻` / `✽` | Процес живий, відповідає одразу |
| `∙` | Процес завершився, але сесію можна продовжити |
| `✢` | `/loop`-сесія спить між ітераціями |

## Клавіатурне керування

| Дія | Клавіша |
| :--- | :--- |
| Peek (останній output) | `Space` |
| Відповісти з peek panel | текст + `Enter` |
| Зайти у повну сесію | `Enter` або `→` |
| Повернутись до таблиці | `←` на порожньому промпті |
| Диспатч + одразу зайти | `Shift+Enter` |
| Закрити Agent View | `/exit` або `/quit` |

## Диспатч нових сесій

Введи prompt у нижньому полі і натисни `Enter` — запускається нова фонова сесія. Кожен Enter — нова сесія (не продовження попередньої).

Особливі форми:

- `/skill-name` — запуск як скіл
- `#1234` або PR URL — якщо сесія вже працює з цим PR, перейде до неї замість нової

## Фільтрація

Введи текст у поле запуску — замість нової сесії відфільтрує список:

| Фільтр | Показує |
| :--- | :--- |
| `a:<name>` | Сесії що запускають конкретного агента |

## Ізоляція файлів

Кожна фонова сесія автоматично переміщається у ізольований git worktree під `.claude/worktrees/` перед першим записом. Паралельні сесії не конфліктують між собою по файлах.

Вимкнути ізоляцію: `"worktree.bgIsolation": "none"` у `.claude/settings.json`.

## Звʼязок з /background

`claude agents` — dashboard для сесій що запущені через [/background](commands/background.md). Також можна запускати нові сесії прямо звідси.
