# 🧠 База знань Claude Code — Інструкції для Gemini

Цей файл містить огляд структури проєкту та обов'язкові правила для взаємодії з цією базою знань Obsidian.

## 📁 Огляд проекту

Цей репозиторій є **базою знань (Knowledge Base)** розробника Java рівня Senior, реалізованою у форматі **Obsidian Vault**. Вона призначена для систематизації технічних знань.

Структура папок:
- `Claude_Code_Knowledge_Base.md` — Головна карта знань (MOC).
- `content/` — Усі технічні нотатки.
- `content/scripts/` — Захисні механізми (хуки).

## 📝 Правила створення та редагування нотаток

При додаванні нових або оновленні існуючих файлів **обов'язково** дотримуйтесь наступних стандартів:

1.  **TL;DR:** Кожна нотатка повинна починатися з розділу `## 📝 TL;DR` (коротке резюме теми, 2-3 речення). Це ЗАВЖДИ має бути найпершим розділом.
2.  **Заголовки:** Заголовок першого рівня (# H1) НЕ використовується всередині файлу (назва файлу є заголовком).
3.  **Навігація:** Нотатка повинна починатися з посилання на головний MOC: `[⬅️](../Claude_Code_Knowledge_Base.md)`.
4.  **MOC-файл:**
    - Усі нотатки реєструються безпосередньо в `Claude_Code_Knowledge_Base.md`.
    - Використовуйте тільки стандартний Markdown для посилань: `[Назва](content/назва.md)`.
    - **Wiki-links `[[...]]` заборонені.**
    - Назви посилань (текст у квадратних дужках) мають бути перекладені українською.
    - Шляхи мають бути відносними та включати розширення `.md`.

## 🤖 Робота Gemini
- **Спілкування з користувачем:** Виключно українською мовою.
- **Технічні завдання:** Пошук, аналіз та генерація коду — англійською мовою (для точності).
