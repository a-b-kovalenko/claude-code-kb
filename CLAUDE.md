# CLAUDE.md

Obsidian vault — база знань про Claude Code для Senior Java-розробника. 46 нотаток: від основ агентської розробки до production-ready workflow, захисних хуків та прикладів субагентів.

## Структура

- `Claude_Code_Knowledge_Base.md` — головний MOC (Map of Content), єдиний реєстр усіх нотаток.
- `content/` — усі технічні нотатки.
- `content/scripts/` — приклади захисних скриптів (bash-guard, flyway-guard).
- `CLAUDE.md` — правила ведення бази знань (цей файл).

## Конвенції нотаток

Кожна нотатка в `content/` **обов'язково**:

1. Починається з навігаційного посилання: `[⬅️](../Claude_Code_Knowledge_Base.md)`. Виняток: нотатки у підпапках (`content/commands/`) — посилання веде до батьківської нотатки-індексу (`[⬅️](../Commands_Reference.md)`), а не до MOC.
2. Має першим розділом `## 📝 TL;DR` (2–3 речення резюме)
3. **Не** містить заголовка H1 — назва файлу є заголовком
4. Реєструється в `Claude_Code_Knowledge_Base.md` у відповідній секції

## Правила посилань

- Тільки стандартний Markdown: `[Назва](content/назва.md)` — wiki-links `[[...]]` заборонені.
- Текст посилань (у квадратних дужках) — українською.
- Шляхи — відносні, з розширенням `.md`.

## MOC та Markdownlint

`Claude_Code_Knowledge_Base.md` використовує `###` заголовки для секцій — **не** Obsidian callouts (`[!abstract]` тощо). Це забезпечує однаковий вигляд в Obsidian і на GitHub.

Хук `.claude/hooks/md-lint.sh` перевіряє кожен записаний `.md` файл **у межах проєкту** і блокує запис при порушенні. Файли поза `$CLAUDE_PROJECT_DIR` (наприклад, памʼять агента) не перевіряються. Потрібен `markdownlint-cli` (`npm install -g markdownlint-cli`).

2. Додати рядок у відповідну секцію `Claude_Code_Knowledge_Base.md`.
