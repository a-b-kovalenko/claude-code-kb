# CLAUDE.md

Obsidian vault — база знань про Claude Code для Senior Java-розробника. 46 нотаток: від основ агентської розробки до production-ready workflow, захисних хуків та прикладів субагентів.

## Структура

- `Claude_Code_Knowledge_Base.md` — головний MOC (Map of Content), єдиний реєстр усіх нотаток.
- `CLAUDE.md` — правила ведення бази знань (цей файл).
- `content/` — усі технічні нотатки.
- `content/assets/` — зображення та медіафайли.
- `content/commands/` — окремі нотатки для команд з нетривіальними параметрами; індекс — `Commands_Reference.md`.
- `content/scripts/` — приклади захисних скриптів (bash-guard, flyway-guard).
- `content/cca/` — розділ підготовки до CCA-F; індекс — `CCA_Foundations.md`.
- `content/cca/qa/` — розбір реальних питань іспиту; індекс — `qa_index.md`.

## Конвенції нотаток

Кожна нотатка в `content/` **обов'язково**:

1. Починається з навігаційного посилання: `[⬅️](../Claude_Code_Knowledge_Base.md)`. Виняток: нотатки у підпапках (`content/commands/`) — посилання веде до батьківської нотатки-індексу (`[⬅️](../Commands_Reference.md)`), а не до MOC.
2. Має першим розділом `## 📝 TL;DR` (2–3 речення резюме)
3. **Не** містить заголовка H1 — назва файлу є заголовком
4. Реєструється в `Claude_Code_Knowledge_Base.md` у відповідній секції

## Правила посилань

- Тільки стандартний Markdown: `[Назва](content/назва.md)` — wiki-links `[[...]]` заборонені.
- Текст посилань (у квадратних дужках) — українською.
- Шляхи — відносні, з розширенням `.md`.

## MOC та Markdownlint

`Claude_Code_Knowledge_Base.md` використовує `###` заголовки для секцій — **не** Obsidian callouts (`[!abstract]` тощо). Це забезпечує однаковий вигляд в Obsidian і на GitHub.

Хук `.claude/hooks/md-lint.sh` перевіряє кожен записаний `.md` файл **у межах проєкту** і блокує запис при порушенні. Файли поза `$CLAUDE_PROJECT_DIR` (наприклад, памʼять агента) не перевіряються. Потрібен `markdownlint-cli` (`npm install -g markdownlint-cli`).

## Мова

- Спілкування з користувачем: **українська**.
- Зміст нотаток: **українська**; код і команди — англійською.

## QA-нотатки (`content/cca/qa/`)

Окремий тип нотатки для розбору питань іспиту. Структура:

1. Backlink: `[⬅️](qa_index.md)` — до індексу QA, не до MOC
2. `## 📝 TL;DR`
3. `## Original` — оригінальний текст питання англійською: **Scenario:**, **Question:**, варіанти **A)–D)**
4. `## Питання` — переклад українською
5. `## Правильна відповідь: X`
6. `## Аналіз варіантів` — розбір кожного варіанту
7. `## Ключові концепції`
8. `## Пов'язані нотатки`

Реєстрація — у відповідній таблиці `qa_index.md`, **не** в MOC.

## Правило MOC для підрозділів

Якщо підрозділ має власний індексний файл (`CCA_Foundations.md`, `qa_index.md` тощо), MOC містить **одне** посилання на цей індекс — не окремі посилання на кожну нотатку підрозділу. Вся детальна навігація — у індексному файлі підрозділу.

## Trailing newline у нових файлах

Після Write завжди забезпечити рівно один trailing newline — MD047 блокує файли без нього:

```bash
printf '\n' >> file.md && perl -i -0pe 's/\n{2,}$/\n/' file.md
```

## Додавання нової нотатки

1. Створити файл у `content/<назва>.md` з дотриманням конвенцій вище.
2. Додати рядок у відповідну секцію `Claude_Code_Knowledge_Base.md`.

## CHANGELOG.md

Сесійний журнал змін у корені проєкту. Один запис на сесію — перед комітом.

Формат запису:

```markdown
## YYYY-MM-DD

### Додано
- **[Назва нотатки](content/назва.md)** — одне речення про що

### Оновлено
- **[Назва нотатки](content/назва.md)** — що саме змінено
```

Найновіший запис — зверху. Оновлювати лише коли є реальні зміни у нотатках.
